How to Run the JHipster Sample Application
=========================================

Prerequisites
------------
1. Java JDK (version specified in pom.xml)
2. Node.js and npm
3. Docker and Docker Compose
4. Maven

Database Setup
-------------
The application uses PostgreSQL which runs in Docker:
1. PostgreSQL database runs on port 5432
2. Database is automatically started with Docker Compose
3. Check if database is running:
   ```
   docker ps
   ```
   You should see a container named "jhipstersampleapplication-postgresql-1"

Starting the Frontend
-------------------
1. Navigate to the project root directory
2. Run the frontend using npm:
   ```
   ./npmw start
   ```
3. The frontend will be available at:
   http://localhost:9001
4. Login page will be at:
   http://localhost:9001/login

Starting the Backend
------------------
1. Navigate to the project root directory
2. Start the Spring Boot application:
   ```
   mvn spring-boot:run
   ```
   or with explicit port configuration:
   ```
   mvn spring-boot:run -Dspring-boot.run.arguments=--server.port=8080
   ```
3. The backend API will be available at:
   http://localhost:8080

Verification
-----------
1. Check if PostgreSQL is running:
   ```
   docker ps
   ```
   Should show PostgreSQL container running on port 5432

2. Check if frontend is running:
   ```
   lsof -i :9001
   ```
   Should show Node.js process

3. Check if backend is running:
   ```
   lsof -i :8080
   ```
   Should show Java process

Troubleshooting
--------------
1. If you see libunistring errors when using Maven wrapper (./mvnw):
   ```
   brew reinstall libunistring
   brew link libunistring --force
   ```
   Or use the Maven binary (mvn) directly instead of the wrapper

2. If the application doesn't start, check:
   - Database connection in application-dev.yml
   - Logs in target/spring-boot.log
   - Docker container status with 'docker ps'
